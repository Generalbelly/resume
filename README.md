#### 2024/10 - 2025/11 / 広告データ利活用プロダクトの新規開発・PdM・エンジニア

溜まった広告データを活用する新規プロダクトの PdM 兼エンジニアとして、企画・設計・開発・サポートを担当。大手顧客へのヒアリングは営業と連携して実施。開発・運用において、生成 AI / LLM を積極的に活用した。

- **広告データ利活用プロダクトの開発（ベータ版リリース）**
  - クリエイティブ制作・案件シミュレーションの課題に対し、顧客ヒアリング・企画・設計・実装・サポートまで一人で担い、以下 3 機能を統合したプロダクトをベータ版として展開した。
    - **案件検索（セマンティック検索）**: 広告運用者が手作業で行っていた類似案件の探索を BigQuery × Vertex AI のベクトル検索で自動化した。
    - **案件シミュレーション**: 類似案件のパフォーマンスデータをもとに新規案件の予測値を算出し、営業・企画のデータドリブンな意思決定を支援した。
    - **クリエイティブ生成**: LP URL から広告バナー画像・テキストを自動生成するシステムを構築した。
- 使用技術: TypeScript, Node.js, Next.js, Cloud Run, PubSub, Memorystore, BigQuery, Vertex AI, Gemini API, OpenAI API

#### 2018/09 - 2022/12 / 広告レポーティングツール テックリード

創業メンバーとしてジョインし、BigQuery を最大限活用した[広告レポーティングツール](https://www.data-be.at/)を CTO の右腕としてゼロから開発。

チーム規模: エンジニア 8〜10 人、デザイナ 0〜1 人、PM/営業 1 人

- **接続広告アカウント数を 3,000 → 20,000 へスケール**
  - プロダクトのスケーラビリティ課題を解決し、広告レポート取得数を約 200,000 件/日まで拡大した。BigQuery のパーティション設計や非同期処理の最適化により実現した
- **広告レポート取得アプリケーションのサーバーレス化（アーキテクチャ刷新）**
  - 既存の GAE モノリスを Cloud Run + PubSub の非同期アーキテクチャに刷新し、スケーラビリティと運用コストの両面を改善した。
  - 新アーキテクチャの導入よって、アカウント登録の当日にデータを提供できる競合優位性を確立した。
  - 使用技術: Go, PubSub, Cloud Run, Datastore, Memorystore, Cloud Scheduler, BigQuery, Cloud Storage
- **データマート構築アプリケーションの開発**
  - 広告データと顧客の予算・KPI データを紐付けるデータマート生成システムを設計・開発した
  - 使用技術: Go, PubSub, Cloud Run, Datastore, BigQuery, Cloud Storage
