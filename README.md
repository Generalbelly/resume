## 強み

**「作る」と「届ける」の両方を担える点**  
スタートアップで創業から携わり、技術選定・設計・実装だけでなく、顧客ヒアリング・要件定義・リリース調整・サポートまで一気通貫で担当してきました。ビジネスインパクトを念頭に優先順位を判断しながら開発を進められます。

**初速と品質のバランス**  
「とりあえず動かす」スピードと、「スケール時に壊れない」設計の両立を意識しています。担当プロダクトでは接続広告アカウント数を 3,000 から 20,000 へスケールさせた経験から、初期設計の重要性を身をもって理解しています。

**AI ツールの実践的な活用と展開**  
LLM を活用した広告プロダクト開発を経験するかたわら、Claude Code を非エンジニア職種（広告運用者）への展開・勉強会開催を自発的に行いました。AI を「作る側」と「広める側」の両方から実践してきた点が強みです。

#### 2024/10 - 2025/11 / AI 活用広告プロダクト新規開発・PdM

新規プロダクトの PdM 兼エンジニアとして企画・設計・開発・サポートを担当。大手顧客へのヒアリングは営業と連携して実施。生成 AI / LLM を積極的に活用し、複数プロダクトを同時推進した。

- **AI 活用広告プロダクト — 顧客ヒアリングから実装・サポートまで一気通貫で担当（ベータ版リリース）**
  - 入札調整の自動化が進む中でも人手に依存し続けるクリエイティブ制作・案件設計の課題に対し、顧客ヒアリング・企画・設計・実装・サポートまで一人で担い、以下 3 機能を統合したプロダクトをベータ版として展開した
  - **案件検索（セマンティック検索）**: 営業が手作業で行っていた類似案件の探索を BigQuery × Vertex AI のベクトル検索で自動化
  - **案件シミュレーション**: 検索で発見した類似案件のパフォーマンスデータをもとに新規案件の予測値を算出
  - **クリエイティブ生成**: LP URL から広告バナー画像・テキストを自動生成
  - 使用技術: Typescript, NodeJS, NextJS, Cloud Run, PubSub, Memorystore, BigQuery, Vertex AI, Gemini API, OpenAI API

#### 2023/01 - 2024/09 / 広告レポーティングツール PdM

組織が PMM / PdM 体制に移行したタイミングで PdM を担当。ビジネスサイドとエンジニアの橋渡しを務めながら、プロダクト推進・既存機能改善・開発体制の整備を担った。

チーム規模: エンジニア 8〜10 人、PMM 1 人、CS 5〜6 人

- **サーバーレスアーキテクチャの全媒体展開（約 40 媒体）**
  - 自ら設計したアーキテクチャをもとに、約 40 媒体へのリプレイスをリードした。アカウント登録と同時に PubSub 経由でデータ取得を即時起動する仕組みを全媒体に展開し、競合が翌日以降まで待たせるところをアカウント登録当日中にデータを提供できる優位性を確立。システム設計から移行計画の策定・進行管理まで主導した
  - 使用技術: Go, PubSub, Cloud Run, BigQuery
- **プロダクトロードマップの策定・管理**
  - CEO・PMM と連携してロードマップを策定し、事業戦略と開発計画の整合を維持した
- **Looker Studio レポート出力の簡易化（レポート生成 約 30 分 → 1 クリック）**
  - Linking API を活用し、顧客が約 30 分かかっていたレポート生成をワンクリックで完了できる仕組みを Google Apps Script で構築。後にこの仕組みはアプリ本体の機能として組み込まれた
  - 使用技術: Google Apps Script, Looker Studio Linking API
- **大手顧客のオンボーディング対応**
  - 大手顧客のオンボーディングに技術側の窓口として参加し、クライアントのエンジニアに広告データのスキーマ・取り込みスケジュール・データマートの活用方法を説明した
- **外部サービス連携ベータ版の開発（GA4・AppsFlyer・Adjust）**
  - 各プラットフォームのデータを BQ に取り込む連携機能を Google Apps Script で実装。部分的な市場ニーズを確認できたが、開発コストとの兼ね合いから本体アプリへの組み込みは見送り。最小工数での検証にとどめつつ、営業が商談で対応媒体として訴求できる材料となった
  - 使用技術: Google Apps Script, BigQuery

創業メンバーとしてジョインし、BigQuery を最大限活用した[広告レポーティングツール](https://www.data-be.at/)を CTO の右腕としてゼロから開発。

  - 使用技術: Go, PubSub, Cloud Run, Datastore, Memorystore, Cloud Scheduler, BigQuery
- **データマート構築アプリケーションの開発**
  - 広告データと顧客の予算・KPI データを紐付けるデータマート生成システムを設計・開発した
  - 使用技術: Go, PubSub, Cloud Run, Datastore, BigQuery

  - 使用技術: Javascript, Vue.js（Vuetify）, PHP（Laravel）, GCE
